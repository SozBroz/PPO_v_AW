I am making a project that uses PPO and NNs to learn AWBW standard play. Right now this is my architecture: What actually runs in trainingSelfPlayTrainer.train() builds sb3_contrib.MaskablePPO with "MultiInputPolicy" and no policy_kwargs (no custom features_extractor). So the trained network is Stable-Baselines3’s default for a Dict observation: a CombinedExtractor that processes each observation key with its default sub-extractor (typically a CNN-style trunk on the spatial box and an MLP on the flat scalars), then fuses them for the actor and critic.The custom class AWBWNet in rl/network.py describes a residual CNN + fusion MLP + policy/value heads design but is not hooked into MaskablePPO in self_play.py, so it is not what training is using unless you add policy_kwargs yourself.Inputs (observation)The env exposes a Dict space 1 filled by encode_state():KeyShapeContents (summary)spatial(30, 30, 59)59 planes: 28 unit (14 types × 2 players), 1 HP, 15 terrain one-hot, 15 property (5 types × neutral/P0/P1).scalars(13,)Funds (2), normalized power bars (2), COP/SCOP active flags (4), turn / MAX_TURNS, active_player, p0_co_id/30, p1_co_id/30, tier encoding.So: feed = board + globals, all float32, normalized as in the encoder docstring.Outputs (action and value)OutputMeaningPolicyLogits over Discrete(ACTION_SPACE_SIZE) with ACTION_SPACE_SIZE = 35_000 (rl/network.py, used by rl/env.py). That’s a single flat index encoding end turn, powers, select/move, attack, capture, wait, load, build, etc. (_action_to_flat / env).MaskingActionMasker sets invalid logits to −∞ so only legal actions get probability (MaskablePPO).ValueOne scalar state-value s)V(s) for PPO’s advantage / return bootstrap.So: output = masked categorical over ~35k discrete actions + one value.PPO loop (context)From self_play.py: n_steps=2048, batch_size=256, n_epochs=10, gamma=0.99, gae_lambda=0.95, etc.—standard MaskablePPO on the vec env.One-line summaryToday: SB3 MultiInputPolicy (default extractors) on spatial + scalars → 35k-way masked policy + scalar value.
