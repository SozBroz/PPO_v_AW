## 5. Hawke COP heal — primary-source-anchored fix

### 5.1 Bug

`engine/game.py` Hawke branch always healed friends `+20` internal HP (`+2` display bars) regardless of whether COP or SCOP fired. Code (pre-fix):

```python
elif co.co_id == 12:
    enemy_loss = 10 if cop else 20
    for u in self.units[player]:
        u.hp = min(100, u.hp + 20)   # BUG: always +20
    for u in self.units[opponent]:
        u.hp = max(1, u.hp - enemy_loss)
```

Per AWBW CO Chart (Tier 1, amarriner.com), COP "Black Wave" heals **+1 HP** (=`+10` internal), not `+2 HP`. The engine over-healed friends by `+10` internal (`+1` display bar) on every Black Wave fire.

### 5.2 Citations

* **AWBW CO Chart** (Tier 1, amarriner.com — repository canon per Phase 11A Kindle precedent):
  https://awbw.amarriner.com/co.php Hawke row —
  - D2D: *"+10% attack to all units."*
  - Black Wave (COP): *"All units gain +1 HP. All enemy units take 1 HP damage."*
  - Black Storm (SCOP): *"All units gain +2 HP. All enemy units take 2 HP damage."*
* **AWBW Fandom Wiki** (Tier 2, supporting): https://awbw.fandom.com/wiki/Hawke
* **Wars Wiki** (Tier 2, vanilla AW cross-check): https://warswiki.org/wiki/Hawke
* **PHP ground truth** (Tier 3): `tools/_phase11j_hawke_cop_drill.py` on gid 1635846 env 30 (Hawke COP "Black Wave" day 16). Own units that took NO combat damage in the same envelope showed clean `+1.0` display HP heals:

  | unit_id  | unit      | pos     | hp before | hp after | Δ display |

  Enemy units not fired upon in env 30 showed `-1.0` display HP baseline (Anti-Air, B-Copter, Infantry, Mech all clean `-1.0`).

  SCOP samples from gids 1617442 / 1635679 confirmed `+2.0` / `-2.0` baseline — engine SCOP behavior was already canonical.

### 5.3 Fix

`engine/game.py:736-771` — make the friend heal symmetric with `enemy_loss`:

```python
elif co.co_id == 12:
    friend_heal = 10 if cop else 20
    enemy_loss  = 10 if cop else 20
    for u in self.units[player]:
        u.hp = min(100, u.hp + friend_heal)
    for u in self.units[opponent]:
        u.hp = max(1, u.hp - enemy_loss)
```

Comment block above the branch documents all four citations and the per-replay PHP heal-delta evidence.

`data/co_data.json` Black Wave description corrected: `"...all own units recover 2 HP"` → `"...all own units recover 1 HP"`. The previous text contradicted both the AWBW chart and the engine's *post-fix* behavior; was inherited folk-rule, not primary-source.

### 5.4 Why no audit-floor regression

The `+10` over-heal on COP made friends healthier in the engine than in PHP. That extra HP carried into the next-turn property-repair pass: engine units ended closer to display HP 10, so the engine *paid less or the same* repair gold than PHP. Net effect on funds: engine-side **surplus or equal**, never deficit — which is why every COP-firing Hawke gid in the corpus was already `ok` pre-fix (the audit only fails on engine-side **shortfalls**). Post-fix, friend HP matches PHP; funds parity is unchanged at every BUILD boundary. **0 regressions across the 936-zip corpus** confirms the analysis empirically.
